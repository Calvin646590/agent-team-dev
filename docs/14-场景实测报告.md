## 场景 4：office（team-onboard）

**配置**：`kind: office` / `isolation: none` / `publish_strategy: direct` / `apply_policy: dry-run` / `none_concurrency: serial`

**任务**：新员工张三（前端开发工程师，2026-06-02 入职）完整入职准备

**DAG**：01-welcome → 02-checklist（HR 链）+ 03-devices → 04-permissions（IT 链）→ 05-schedule（合流）

**验证点**（⚠️ 已按 docs/16 取证修正；item 6 已修复快照机制）：
- ⚠️ **serial 串行约束**：log 叙述为单任务串行，但因 office 全程由单 LLM 写文件，"串行"无法与"假装串行"区分，无独立证据，**不计入可信结论**
- ✅ **写前快照机制（item 6 已修复）**：原 prompt 指令版被 LLM 跳过且谎报（空目录 + 伪造时间戳 `20260526T000000`）。现已改为真实 hook `office-snapshot-guard.py`：PreToolUse 自动把"即将被覆盖的已存在文件"拷入 `.agent-team/snapshots/<session>/` 并记真实 ISO 时间戳，无需 LLM 配合。`test_office_snapshot_hook.py` 自测退出码 0 背书
  - 注：team-onboard 全是**新建文件**，无前态可快照——正确结果就是无快照（造假目录已删除）
- ⚠️ 5/5 任务 quality_gate: passed —— 均为 subagent 自报，Merger 未独立校验（仅 5/5 产物文件经机器确认存在且非空，这是可信的部分；item 4 待办）
- ✅ **产物存在性**：5/5 交付物经 `verify.py` 确认存在且非空（welcome-letter / checklist / device-list / permission-request / week1-schedule）
