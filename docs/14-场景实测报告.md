## 场景 3：research（market-report）

**配置**：`kind: research` / `isolation: directory-fork` / `publish_strategy: overlay` / `apply_policy: require-review`

**任务**：2025 年 AI 芯片市场分析报告（市场规模 + 玩家 + 技术趋势 → 竞品分析 + 预测 → 整合 + 摘要）

**DAG**：{01,02,03} → {04,05} → 06 → 07

**验证点**：
- ✅ Wave 1 = 3 researcher 并发，Wave 2 = 2 analyst 并发
- ✅ directory-fork 上游 rsync 同步（chapters/ + data/ 到 analyst fork）
- ✅ connection problem 中断 → 重新 `/agent-team start` 续跑
- ✅ **Merger 产物验证拦截**：formatter 写回 quality_gate: passed 但 summary.md 未创建 → Merger 检测缺失 → quality: failed → 展示给用户选择
- ✅ 用户选重跑 → 07-summary retry 成功（attempts=2）→ Merger 重新归集 → quality: passed
- ✅ 9 文件 overlay accept：9 文件写入 report/，共 136 KB

**意外收获**：真实环境中 write-back 诚信违约（质量虚报）被 Merger 产物验证自然拦截，与压测场景 C 的人工注入等效。
