## 助手推荐：T4 收尾形态

**结论**：PR-style 差异 + 项目级默认行为 + 用户可临时覆盖

**推理链**：
1. ADR-0014 选了 Worktree 隔离 → 合并必然产生 diff
2. 把 diff 当作"PR 候选"展示给用户审核，比直接 commit 安全得多
3. 不破坏用户既有 git 工作流（如果用户用 GitHub PR 流程，可直接转化）
4. 但每次都让用户审会拖慢节奏 → 项目 README 里声明默认行为：
   - `auto-merge`：直接合并到目标分支（高信任场景）
   - `require-review`：每次必审（默认）
   - `dry-run`：完全不动主分支，仅生成 diff 报告
5. 用户在对话里可临时覆盖默认（"这次直接 merge 吧"）

→ 形成 ADR-0015

## 推导出的工作流（场景 B 完整版）

```
T0 用户启动
  ├─ 在项目目录里说"加个邮件提醒功能"
  └─ Plugin 检测到 agents/ 目录，激活 team

T1 团队组建
  ├─ Coordinator 调用 RequirementGatherer 追问需求（提前多久？哪种邮件服务？...）
  ├─ 需求明确后，Coordinator 走两级派生：
  │   ├─ 规则匹配器查 agents/README.md 里的 task→role 映射
  │   └─ 模糊场景降级到 Coordinator 自己用 LLM 判断
  └─ 派生：dba + backend + frontend + tester（举例）

T2 任务拆解
  ├─ Coordinator 创建 .agent-team/tasks/{01-db-migration,02-email-service,03-api,04-ui,05-tests}.md
  ├─ 每个 md 包含：标题、负责 agent、依赖、验收标准、状态
  └─ Coordinator 维护 .agent-team/tasks/index.md（全局视图）

T3 并行执行
  ├─ 派生引擎为每个业务 agent 创建独立 git worktree（isolation: "worktree"）
  ├─ agent 在自己的 worktree 里干活，更新自己的 tasks/<id>.md
  ├─ 有依赖的子任务（API 依赖 DB schema）→ Coordinator 调度顺序，
  │  或被依赖 agent 完成后通过 handoff_to 直接调用下游
  └─ 卡住的 agent 通过 escalate_to 上报 Coordinator

T4 收尾
  ├─ 所有 agent worktree 完成 → 合并 agent 把多个 worktree 合到一个临时分支
  ├─ 临时分支跑测试（tester agent 验证 quality_gates）
  ├─ 生成 PR-style 差异 + 变更说明
  └─ 按项目默认 / 用户即时指令决定：auto-merge / require-review / dry-run
```

## agent md schema 草案

### 框架层 agent (plugin 内置)

```markdown
---
name: coordinator
description: 接收任务、派生决策、调度子任务、合并产物
tools: [Agent, Read, Write, Bash]   # Claude Code 原生
model: opus                          # Claude Code 原生
role: framework                      # 我们的扩展：framework | business
---

[system prompt：协调者的工作风格、判断原则、何时降级使用 LLM 判断、如何写 tasks/ index]
```

### 业务层 agent (项目 agents/ 定义)

```markdown
---
# Claude Code 原生字段
name: backend-developer
description: 负责 Next.js API routes 与数据库交互
tools: [Read, Write, Edit, Bash]
model: opus

# team 扩展字段
role: business
capabilities:
  - 实现 REST API
  - 编写数据库查询
  - 集成外部服务（邮件、支付等）
files_scope:
  read: ["src/**", "schema.prisma", "package.json"]
  write: ["src/api/**", "src/lib/**", "prisma/migrations/**"]
handoff_to: [tester]
escalate_to: coordinator
triggers: ["后端", "API", "数据库", "服务端"]
quality_gates:
  - 所有相关测试通过
  - lint 与 type-check 无错误
worktree: true
---

[system prompt：这个 agent 的工作风格、技术偏好、约束条件、不允许做的事]
```

## 项目 README schema 草案

```markdown
# Project Name

## team-config

```yaml
on_complete:
  default: require-review        # auto-merge | require-review | dry-run
  branch: main                   # 目标合并分支

derivation_rules:                # 任务模式 → 角色（规则匹配器用）
  - pattern: "API|后端|数据库"
    roles: [backend-developer, dba]
  - pattern: "UI|界面|前端"
    roles: [frontend-developer]
  - pattern: ".*"                # 兜底
    fallback: coordinator         # 无法匹配 → 协调者降级
```
```
