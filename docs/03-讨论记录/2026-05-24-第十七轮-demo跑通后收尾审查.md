## Follow-up #1 处理（2026-05-24，同轮）

### 根因分析

调查后发现：
- `commands/*.md`（legacy format）和 `skills/<name>/SKILL.md`（现代格式）官方说"加载完全等价"
- 官方 plugin.json 同样没有 `commands` 显式声明字段——不是 plugin.json 缺字段的问题
- coordinator/scheduler/mediator skills **已成功加载**（demo 验证），证明 skills/ 格式对 local-desktop-app-uploads plugin 可用
- 根因假设：`commands/` legacy format 对 local-desktop-app-uploads 类型 plugin 可能未实现 slash command 注册；skills/ 格式已验证

### 修复方案

**新增 `agent-team/skills/team/SKILL.md`**，frontmatter 加 `name: team` + `argument-hint`（两者触发 slash command 注册）：
- 内容与 `commands/team.md` 完全一致（强制框架协议不变）
- `commands/team.md` 保留作 legacy fallback

**plugin 版本升至 v0.1.2**，重新打包 agent-team.zip，同步到已安装目录。

### 验证结果 ✅

`~/.claude/commands/team.md` 方案**成功**：
- `/team`（无参数）正确触发，展示用法菜单
- skill 注册在 system-reminder 中可见：`team: agent-team 强制入口...`

### 根因（最终确认）

| 路径 | 结果 | 说明 |
|------|------|------|
| plugin `commands/team.md` | ❌ Unknown command | plugin commands/ 目录不会注册用户可输入 slash command |
| plugin `skills/team/SKILL.md` + argument-hint | ❌ Unknown command | plugin skills/ 只触发 LLM-context skill，不触发用户输入的 slash command |
| `~/.claude/commands/team.md` | ✅ 成功 | 全局用户命令路径，Claude Code 原生支持 |

**结论**：plugin.json 插件系统的 commands/ 和 skills/ 都只供 LLM-triggered 使用；用户可输入的 slash command 必须放在 `~/.claude/commands/`（全局）或 `<project>/.claude/commands/`（项目级）。

### 安装说明更新

使用 agent-team 需要额外一步：把 `commands/team.md` 复制到 `~/.claude/commands/team.md`。已写入运行手册。
