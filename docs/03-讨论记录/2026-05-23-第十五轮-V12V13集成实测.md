# 2026-05-23 第十五轮：V-12 / V-13 集成实测（消化高级审查 R-3）

> 高级审查 R-3 指出 V-1~V-11 只验了单点原语、集成未验。本轮用 plugin v0.0.6 实测两个集成未知，结果**都比预期乐观**。

## V-12：worktree 并发 + 跨 worktree 同步 —— ✅ 通过

**测法**：test-worktree-sync skill 三步——① 并发调 worker-wt-a/b（各 isolation:worktree 提交一文件）② 主会话合并上游分支回 base ③ 合并后调 worker-wt-c 看下游能否见上游。

**实测结果**：
- **步骤 1**：A 在 `worktrees/agent-a1d779…` 分支 `worktree-agent-a1d779…` 提交 `out-a.txt`；B 在 `agent-adc45…` 提交 `out-b.txt`。两者**并发 + 各自独立 worktree/分支**
- **步骤 2**：主会话 `git worktree list` / `git branch -a` 看得到 A/B 分支；`git merge`（octopus 一次合两分支）成功，out-a.txt + out-b.txt 落进 main
- **步骤 3**：worker-wt-c 在 `agent-ac4badf…` worktree（从更新后 main 派生），`ls out-a.txt out-b.txt` **都看得到**，内容完整，git log 含 merge commit

**结论**：git-worktree 同步**可行**，但模型要修正——
> 不是"把上游 rebase 进一个已存在的下游 worktree"（Claude Code 自动建 worktree，塞不进去），
> 而是"**上游分支先 merge 进 base，下游 subagent 的 worktree 在 spawn 时从更新后的 base 派生**，自动包含上游产物"。

→ 写入 ADR-0042，修正 ADR-0018 的 sync 表述、02 IsolationStrategy 表、scheduler.md。
（推翻了审查时"下游可能看不到上游"的悲观预测——开发场景 T3 的核心机制成立。）

## V-13：PreToolUse hook 能否识别"当前哪个 subagent" —— ✅ 通过

**测法**：hook 升级为 dump 完整 payload；test-hook-identity skill 让主会话发一个标记 Bash、再让 worker-hook subagent 发一个标记 Bash，对比两条 payload。

**实测结果**：
| 字段 | 主会话调用 | worker-hook（subagent）调用 |
|------|-----------|----------------------------|
| session_id | 相同 | 相同（不能区分） |
| cwd | 相同 | 相同（不能区分） |
| **agent_id** | ❌ 无 | ✅ `a61961bbef6f3a937` |
| **agent_type** | ❌ 无 | ✅ `worker-hook` |

- hook **对 subagent 的工具调用也触发**（payload 被 dump 下来）
- payload 含**直接身份字段** `agent_id` + `agent_type`，**只在 subagent 调用时出现**，主会话调用时没有

**结论**：strict 模式 **per-agent files_scope 可行**——hook 读 `agent_type`，加载该 agent 的 `.claude/agents/<agent_type>.md` 的 files_scope.write，越界 deny。无此字段 = 主会话发起。
→ 写入 ADR-0043，补强 ADR-0019。

## 累计实测确立的机制（截至本轮）

1. 框架层 = 3 skill（主会话）+ 2 subagent（ADR-0032）
2. 主会话可并发派多 subagent + 真并行（V-10）
3. skill 可接力 skill（V-11）
4. worktree 同步 = 上游合 base、下游从更新后 base 派生（V-12 / ADR-0042）
5. strict per-agent scope = hook 读 payload.agent_type（V-13 / ADR-0043）
6. 共享文件 `.agent-team/workspace/` 双向通（V-7）
7. 6 条 plugin/Claude Code 运行约束（见第十四轮）
