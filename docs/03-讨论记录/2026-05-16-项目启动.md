# 2026-05-16 项目启动讨论

> 第一次沟通，目标：定义项目方向、建立文档结构

## 用户初始诉求

> 我们现在开始设计一个通用的 agent team，先沟通定义项目，把项目理解清楚了我们再动手写代码。现在的工作内容就是讨论项目，但是需要把我们的沟通制作成 md 文件，保证沟通开发有痕迹，事后可以回溯。

**关键约束**：
- 工作模式 = 先讨论清楚再动手
- 所有沟通必须沉淀到 md 文件
- 留痕、可回溯

## 助手方案

提议文档结构：
```
docs/
├── 00-项目愿景.md
├── 01-需求定义.md
├── 02-架构设计.md
├── 03-讨论记录/    ← 原始对话
└── 04-决策日志.md  ← ADR 风格
```

## 第一轮提问与回答

| 问题 | 回答 |
|------|------|
| 主要使用场景？ | **全部** —— 办公/开发/创作/分析。通用 agent team，每个 agent 在具体项目上的职责由项目 md 文件设定 |
| 运行形态？ | **在 Claude Code Desktop 里运行** |
| 技术栈？ | **自研编排框架** |
| 团队规模与角色分工？ | **可动态扩展的 agent 池** |

## 提炼出的核心特征

1. **通用 (Universal)** —— 不绑定垂直场景
2. **配置驱动 (Config-driven)** —— 项目级 md 文件定义 agent 行为，类比 Claude Code 的 CLAUDE.md 但放大到团队级
3. **寄宿在 Claude Code Desktop** —— 复用已有工具生态、模型、UI
4. **动态 agent 池** —— 按需派生，不固定角色
