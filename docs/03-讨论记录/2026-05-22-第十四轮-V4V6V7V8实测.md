## V-4：subagent 在独立 worktree 里运行 —— ✅ 通过

**测法**：worker-wt 声明 `isolation: worktree`，由 test-worktree skill 用 Agent 工具调起，对比主会话与 worker 的 pwd/分支。

**实测结果**：
- worker-wt 的 pwd = `.../test-project/.claude/worktrees/agent-aa31d617e5e75ac0f`（独立 worktree 目录）
- worker-wt 的分支 = `worktree-agent-aa31d617e5e75ac0f`（非 main）
- `git worktree list` 同时列出主 worktree（main）+ 独立 worktree（locked）
- worker 写的 wt-marker.txt 落在独立 worktree 内，不污染主树

**结论**：`isolation: worktree` 生效，开发场景 IsolationStrategy.git-worktree 的底座成立。

## V-6：PreToolUse hook 拦截 —— ✅ 通过

**测法**：plugin v0.0.5 带 `hooks/hooks.json`（matcher: Bash）+ `pretooluse-check.sh`（含 STRICT_TRIPWIRE 即 deny）。test-hook skill 先跑普通命令、再跑 tripwire 命令。

**实测结果**：
- 普通命令 `echo "normal command ok"` 正常返回（hook 不误伤）
- 含 STRICT_TRIPWIRE 的命令被拦截，返回 `V-6 HOOK FIRED: agent-team PreToolUse 拦截了含 STRICT_TRIPWIRE 的命令`

**结论**：PreToolUse hook 可注册可拦截，**strict 模式 files_scope 强制（ADR-0019）可行**。
**副产发现**：`hooks/hooks.json` **自动发现**即生效，没在 plugin.json 显式声明 hooks 字段。延续了"manifest 少声明更稳"的规律（对比第十轮 `agents: ["agents"]` 反而把注册搞砸）。

## V-7：主会话 ↔ subagent 共享文件 —— ✅ 通过

**测法**：test-shared-file skill 写 `.agent-team/workspace/from-main.txt`（带时间戳 marker）→ 调 worker-io 读它并写 from-worker.txt → 主会话读回。

**实测结果**：
- 主会话写入 `MAIN-MARKER-1779430345 来自主会话`

**结论**：主会话↔subagent 经 `.agent-team/workspace/` 双向通信链路完整，**ADR-0008 的"共享文件"通信底座成立**。

## 累计的运行约束 / 经验（实测得来）

1. plugin manifest 字段**少声明更稳**：agents/skills/hooks 都靠目录自动发现；显式声明易踩坑（第十轮 `agents:["agents"]` 把注册搞砸）
2. plugin manifest 必须在 `.claude-plugin/plugin.json`
3. 项目级 subagent 按**会话项目根**注册，不是 Bash cwd（第十三轮）
4. subagent 拿不到 Agent 工具，不能派 subagent（第十轮，ADR-0032 的根据）
5. skill 加载到主会话后保留 Agent + Skill 工具，可派 subagent、可接力 skill
6. 装/改 plugin 后需 cmd+Q 完全重启 Desktop 才注册
