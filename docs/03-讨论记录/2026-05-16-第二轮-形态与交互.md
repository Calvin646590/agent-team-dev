## 关键架构决定

### 1. 载体 = Claude Code Plugin
打包 skills + agents + slash commands + (可能的) hooks，用户一键启用。

### 2. 入口 = 多入口共存
- 自然语言：plugin 内 skill 的 description 触发
- 项目内：进入含 `agents.md`（或类似配置）的目录自动激活
- 斜杠命令：`/team <action>` 显式调度

### 3. 用户模式 = 可切换
- 指挥官：用户主导每一步分工与决策
- 旁观者：team 自主运转，用户在检查点审核
- 同事：用户作为团队一员，被 @ 求助 / 主动接手某环节

### 4. 通信 = 父子调用 + 共享文件
- 父→子调用传递任务、获取结果（复用 Claude Code Agent 工具机制）
- 共享文件承载中间产物、跨 agent 状态、长上下文
