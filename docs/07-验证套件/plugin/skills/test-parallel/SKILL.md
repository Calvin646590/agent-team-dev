# V-10 验证：主会话并发调多个 subagent + 收敛

你（**主会话**）现在被 `test-parallel` skill 加载。本次唯一任务：**验证主会话能不能一次并发派多个项目级 subagent，并把它们的返回全部可靠收回**。

这是架构 T3"无依赖任务并行"的命脉。

## 必须完成，原样报告（不解读、不兜底）

### 步骤 1：确认靶子在
用 Bash 跑 `ls .claude/agents/`，确认 `worker-1` / `worker-2` / `worker-3` 三个都在。原样贴。

### 步骤 2：并发派三个 worker（关键！）
**在同一条消息里，同时发起 3 个 Agent 工具调用**（subagent_type 分别为 `worker-1`、`worker-2`、`worker-3`），不要一个一个串行调。
每个 worker 会 sleep 2 秒再返回，用于区分并发与串行：
- 真并发 → 总耗时 ≈ 2 秒
- 串行 → 总耗时 ≈ 6 秒

### 步骤 3：报告每个 worker 的原始返回
把 worker-1 / worker-2 / worker-3 各自返回的字符串原样贴出来（应各含自己的编号 + PID + 时间戳）。

### 步骤 4：结论
回答：
- **三个 worker 都成功返回了吗？** 是 / 否（缺哪个）
- **返回内容能区分是哪个 worker 吗？**（PID/编号是否各不相同）是 / 否
- **从耗时/时间戳看，是真并发还是串行？** 并发(≈2s) / 串行(≈6s) / 看不出
- **收敛是否可靠？**（有没有丢返回、串台、错配）可靠 / 有问题（说明）

## 判定
- 三个都回 + 可区分 + 收敛可靠 → **V-10 通过**（并发是否真并行是加分项，串行也可接受但要记录）
- 丢返回 / 串台 / 调不动多个 → **V-10 不通过**，需重新设计 T3 调度（如退化为串行 + 进度文件）

## 禁止
- **不要串行调**（那测不出并发）—— 必须同一条消息里 3 个 Agent 调用
- 不要派一个 subagent 让它去调 worker（subagent 没有 Agent 工具，调不动）
- 不要自己模拟 worker 返回
