# 项目愿景

> 状态：初稿（2026-05-16） · 来源：[首次启动讨论](03-讨论记录/2026-05-16-项目启动.md)

## 一句话定义

构建一个**通用的、可动态扩展的 agent 团队**，寄宿在 Claude Code Desktop 中，通过项目内的 md 配置文件来适配不同场景的工作流。

## 为什么做

单个 agent 在面对复杂任务时存在天然瓶颈：上下文有限、视角单一、无法并行。多 agent 协作可以突破这些限制，但目前主流方案要么绑死特定场景（比如只做编码），要么需要为每个项目重新搭建一套。

我们想要的是：**一套底层框架 + 项目级配置 = 任意场景的 agent 团队**。

## 核心特征

### 1. 通用 (Universal)
不绑定具体业务领域。同一套底层能力，既可以服务办公事务、也可以做软件开发、内容创作、数据分析、研究调研等任何工作。

### 2. 配置驱动 (Config-driven)
agent 在每个项目上的具体职责、协作方式，**不是写死在框架里**，而是由该项目目录下的 md 配置文件来定义。

类比：Claude Code 用 `CLAUDE.md` 配置单 agent 行为 → 我们把这套思路放大到团队级别，用一组 md 文件配置整个团队的角色、协作规则、工作流。

### 3. 寄宿在 Claude Code Desktop
不另起炉灶做新的 runtime。复用 Claude Code 已经成熟的：
- 工具生态（文件操作、Bash、Web、MCP 等）
- 模型调用
- 桌面端 UI 与会话管理
- 权限模型

### 4. 可动态扩展的 agent 池
agent 数量不固定。框架提供"派生 agent"的能力，按任务需要动态创建、复用、销毁。

## 不做什么（明确边界）

- ❌ **不做独立的 runtime / 桌面应用** —— 寄宿在 Claude Code Desktop 内
- ❌ **不绑定特定业务场景** —— 通用性优先，避免为某一垂直场景做特殊优化
- ❌ **不重复造工具轮子** —— 工具能力借助 Claude Code 已有生态（包括 MCP）
- ❌ **不依赖第三方编排框架**（LangGraph、AutoGen 等）—— 自研编排，保持轻量与可控

## 关键问题已解答（截至 2026-05-16 第六轮讨论）

| 原始问题 | 答案 | 出处 |
|---------|------|------|
| 配置 md 最小集合 | `README` (含 team-config) + `.claude/agents/<name>.md` × N（必需） | ADR-0009, 0023 |
| 寄宿技术形态 | Claude Code Plugin（3 个框架层 skill + 2 个框架层 subagent + 可选 slash command + 可选 hooks） | ADR-0005, 0032 |
| 与 Claude Code Agent 的关系 | 补充而非替代：复用 Claude Code Agent 工具调用、worktree 隔离机制；自研薄编排层在其上 | ADR-0003, 0014 |
| 动态派生触发 | 多级：规则匹配 → LLM 兜底 → 决策缓存 | ADR-0010, 0026 |
| agent 通信 | 父子直接调用 + 共享文件（`.agent-team/`） | ADR-0008 |
